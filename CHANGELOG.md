# Changelog

## [Unreleased]

- No unreleased changes yet.

## [0.1.0] - 2026-07-02

### Added

- Initial Windows tray app for Codex rate-limit monitoring.
- WPF dashboard with 5-hour and weekly reset windows.
- Read-only Codex app-server integration.
- Reset credit count display.
- Configurable low-remaining alerts, manual reset credit notifications, and direct reset detection.
- In-app notification center with a title-bar bell, unread badge, and compact alert history.
- Persisted global notification mute in the tray menu.
- Optional per-user Start with Windows setting.
- Experimental opt-in reset-credit expiry metadata with tray/dashboard status, an all-credit expiry timeline, expiry warnings, and strict fallback to official app-server counts.
- Smart auto-refresh cadence: 5 minutes normally, 1 minute near reset, and 30 seconds when a reset is imminent.
- Branded multi-resolution application/window icon generated by `packaging/generate-app-icon.ps1`.
- Core parser, redaction, tray icon, notification, startup, refresh cadence, and packaging regression tests.
- GitHub issue templates, pull request template, release badges, and tag-driven release automation.

### Changed

- Redesigned the dashboard into a modern frameless Windows 11 visual system with custom rounded chrome, a gradient-mesh backdrop, twin radial usage gauges, monospace data readouts, and staggered load animation.
- Rebuilt the system tray icon into a dual-signal multi-resolution gauge: 5-hour outside, weekly inside, with compact 16 px taskbar rendering.
- Unified status colors across the dashboard and tray into one usage-state ramp; blue is reserved for the weekly window.
- Improved tray hover text and tray menu reset details, including exact local reset times.
- Polished public README, privacy notes, repository ignore rules, line-ending policy, and release hygiene for the first open-source push.

### Fixed

- Hardened refresh, docs-link, and exit handling so transient failures surface in the UI instead of crashing the resident app.
- Hardened Codex CLI subprocess startup so the app resolves an absolute `codex` path from trusted `PATH` entries and preserves Windows command shims safely.
- Ensured reset notifications follow the intended semantics: direct limit resets require remaining capacity to reach 100%, and reset-credit notifications require the available credit count to increase.
